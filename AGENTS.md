# 🤖 Інструкція для ШІ-агентів (CraftLLC)

Якщо ти ШІ-агент, що працює з цим репозиторієм, ти **МУСИШ** суворо дотримуватися цих архітектурних та технічних правил.

## 📁 Структура проекту та Сторінки

1.  **Чисті URL-адреси:** Кожна сторінка повинна мати власну директорію, всередині якої знаходиться `index.html`.
    *   ✅ Правильно: `/account/index.html` (доступно як `domain.com/account`)
    *   ❌ Неправильно: `/account.html`
2.  **Статичні ресурси:** Увесь статичний контент (CSS, JS, зображення, шрифти) має зберігатися ВИКЛЮЧНО в папці `/static/`.
    *   `/static/css/` — стилі
    *   `/static/js/` — скрипти
    *   `/static/img/` — зображення

## ⚙️ Бекенд та API (Cloudflare Workers)

1.  **Розподіл обов'язків:** `_worker.js` відповідає за API, автентифікацію та динамічний контент.
2.  **Ніяких сторінок у коді:** Ніколи не вставляй HTML-код сторінок безпосередньо в `_worker.js`. Бекенд має повертати:
    *   JSON-відповіді для API.
    *   Статичні файли через `env.ASSETS.fetch(request)`.
    *   Динамічні "знання" для CraftAI за адресою `/api/craftai/knowledge`.

## 🔒 Безпека та Стандарти

1.  **Паролі:** Використовуй PBKDF2 з унікальною сіллю (`salt`) для кожного користувача.
2.  **Порівняння:** Завжди використовуй `constantTimeCompare` для перевірки хешів та токенів, щоб запобігти timing-атакам.
3.  **Cookies:** Використовуй прапорці `HttpOnly`, `Secure`, `SameSite=Lax`.
4.  **CORS:** Обмежуй доступ тільки перевіреним доменам (наприклад, `https://craftllc.pages.dev`).

## 🎨 Дизайн та UX

1.  **Стилі:** Дотримуйся існуючої темної теми (#141414) та акцентного помаранчевого кольору (#ffa516).
2.  **Лоадери:** Кожна сторінка повинна мати `#loader-wrapper` для плавного завантаження.
3.  **Cookies Consent:** Банер згоди має відображатися на кожній сторінці через глобальний `static/js/main.js`.
